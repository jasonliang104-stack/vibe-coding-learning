## 4. 具体安装步骤

## 4.1 安装 pnpm

你当前机器没有 `pnpm`，先装它。

执行：

```bash
npm install -g pnpm
```

安装完成后验证：

```bash
pnpm -v
```

---

## 4.2 安装 Docker Desktop

虽然第一版部署方案不是“先走容器化”，但本地开发很适合用 Docker 带数据库和缓存，后面会轻松很多。

建议安装：

```bash
brew install --cask docker
```

安装后：

1. 手动打开 `Docker`
2. 等待 Docker Desktop 完全启动

验证：

```bash
docker --version
```

---

## 4.3 安装 MySQL

建议本地直接装一个 MySQL 8。

执行：

```bash
brew install mysql
brew services start mysql
```

验证：

```bash
mysql --version
```

首次初始化后，建议创建项目数据库：

```sql
CREATE DATABASE resume_optimization CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

推荐本地连接信息先统一为：

- host: `127.0.0.1`
- port: `3306`
- database: `resume_optimization`

---

## 4.4 安装 Redis

执行：

```bash
brew install redis
brew services start redis
```

验证：

```bash
redis-server --version
redis-cli ping
```

如果返回：

```bash
PONG
```

说明 Redis 已可用。

---

## 4.5 升级 Python 到 3.11+

你当前机器是 `Python 3.9.6`，能用，但我不建议拿它做新项目。

建议安装 `Python 3.11`：

```bash
brew install python@3.11
```

安装后确认：

```bash
/opt/homebrew/bin/python3.11 --version
```

如果你的机器不是 Apple Silicon，路径可能不同，可用：

```bash
which python3
ls /usr/local/bin/python3*
ls /opt/homebrew/bin/python3*
```

后续 AI 服务建议固定用 `python3.11` 创建虚拟环境，不直接用系统自带 Python。

- NestJS 主业务服务一个仓库

如果你想最省事，建议先沿用你现在的桌面目录方式。

## 6. 推荐环境变量约定

后续三个项目都建议各自维护自己的 `.env`。

### 6.1 NestJS 主业务服务

建议最少包含：

```env
NODE_ENV=development
PORT=3000
DATABASE_URL=mysql://root:password@127.0.0.1:3306/resume_optimization
REDIS_URL=redis://127.0.0.1:6379
JWT_SECRET=replace_me
COS_SECRET_ID=replace_me
COS_SECRET_KEY=replace_me
COS_BUCKET=replace_me
COS_REGION=replace_me
WECHAT_APP_ID=replace_me
WECHAT_APP_SECRET=replace_me
WECHAT_PAY_MCH_ID=replace_me
WECHAT_PAY_API_KEY=replace_me
```

REDIS_URL=redis://127.0.0.1:6379

MODEL_GATEWAY_API_KEY=replace_me
