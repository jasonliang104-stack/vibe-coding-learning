## 2. 接口设计原则

### 2.1 基础原则

- 接口风格：`REST 为主，少量任务型接口按业务动作命名`
- 版本号：统一使用 `v1`
- 鉴权方式：`JWT / Token`
- 返回结构：统一返回结构
- 文件下载：后端生成下载地址，前端下载
- 任务机制：异步任务 + 前端轮询

### 2.2 统一前缀建议

统一前缀建议为：

`/api/v1`

示例：

- `/api/v1/auth/wechat-login`
- `/api/v1/resumes`
- `/api/v1/tasks/{taskId}`

### 2.3 统一响应结构建议

建议统一为：

```json
{
  "code": 0,
  "message": "ok",
  "data": {}
}
```

可按需补充：

- `requestId`
- `success`

### 2.4 鉴权说明

默认分为两类接口：

- 无需登录接口
- 需要登录接口

需要登录的接口统一要求携带业务 token。

## 4. 接口清单初稿

## 4.1 认证与用户

### 4.1.1 微信登录

**POST** `/api/v1/auth/wechat-login`

用途：

- 小程序登录态换取业务 token
- 首次登录时创建用户

请求建议：

- `code`
- `userInfo`（如有）

响应建议：

- `token`
- `userId`
- `isNewUser`
- `newUserBonusPoints`（如本次为新注册用户且赠送了体验积分）
- `profile`

是否鉴权：否

---

### 4.1.2 获取当前用户信息

**GET** `/api/v1/users/me`

用途：

- 获取当前登录用户基础资料

响应建议：

- `userId`
- `nickname`
- `avatarUrl`
- `membershipStatus`
- `pointBalance`
- `originalResumeCount`
- `optimizedResumeCount`
- `taskCount`

是否鉴权：是

---

### 4.1.3 更新用户基础信息

**PATCH** `/api/v1/users/me`

用途：

- 更新昵称、头像等基础信息（如后续需要）

是否鉴权：是

## 4.5 任务与优化流程

- `resumeId` 或 `resumeVersionId`

## 8. 当前结论

第一版接口设计遵循以下方向：

- `/api/v1` 统一版本前缀
- REST 为主，任务动作接口补充业务命名
- 统一返回结构
- 小程序通过 token 调用业务接口
- 任务流程采用创建任务 + 轮询状态
- 下载采用临时签名地址方案
- 会员、积分、视频奖励、支付形成独立但可协同的接口域

本文档作为接口设计清单初稿，下一步建议继续做：

1. 按接口分组细化请求/响应字段
2. 输出 Swagger 风格接口明细版
3. 补全错误码与状态枚举设计
